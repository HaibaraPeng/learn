## 1. 项目概述

### 1.1 项目定位

**OpenCode 是什么？**
一个 AI 驱动的开发工具，类似于 Cursor、Windsurf，让开发者在终端和 IDE 中与 AI 协作编程。

**WHY 需要这个项目？**

- 解决 AI 编程工具碎片化问题（一个工具支持多模型、多平台）
- 提供本地优先的开发体验（数据隐私、离线可用）
- 支持企业级部署（自托管、权限控制）

**技术选型核心：**

- **运行时：** Bun（高性能 JS 运行时）
- **前端框架：** SolidJS（响应式、无虚拟DOM）
- **AI SDK：** Vercel AI SDK（统一多模型接口）
- **数据库：** SQLite + Drizzle ORM
- **UI组件：** Kobalte + Tailwind CSS

## 7. 关键技术决策

### 7.1 为什么用 Bun 而不是 Node.js？

1. **性能优势**：启动快、HTTP 服务器快
2. **内置能力**：自带 test、bundle、SQLite
3. **原生 TS 支持**：直接运行 .ts 文件
4. **更好的 API**：Bun.file()、Bun.serve()

### 7.2 为什么用 SolidJS 而不是 React？

1. **性能**：无虚拟DOM、编译时优化
2. **响应式**：细粒度更新、无重渲染
3. **包体积**：运行时更小
4. **心智模型**：更像原生 JS、学习曲线平缓

### 7.3 为什么用 Effect？

1. **类型安全错误**：编译时知道所有可能的错误
2. **资源管理**：自动清理、Scope 机制
3. **并发控制**：Fiber、取消、超时
4. **可观测性**：内置日志、追踪
